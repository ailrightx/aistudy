02 · 驱动力分析

关键洞察：KV Cache是最凶猛的单项因素

KV Cache随上下文长度线性增长，随并发数线性增长，两者均在快速增加，形成乘法效应。128K上下文×100并发=1.6TB KV Cache，全部需要在HBM中实时读写——这是DDR物理带宽无法满足的需求，不是成本问题，是物理约束。

HBM vs LPDDR：增长本质的不对称性

# 内存需求增长函数对比 HBM需求 = f(参数量 × 上下文长度 × 并发数 × 推理深度) → 多变量乘法关系 → 真正的指数增长 LPDDR需求 = f(专家offload量 + 系统调度开销) → 加法关系为主 → 快速线性增长 ## 带宽要求对比（物理约束） H100 HBM3: 3.35 TB/s ← 模型推理所需带宽量级 DDR5 CPU: ~0.3 TB/s ← 差距10×以上，无法替代

关键发现：KV Cache > 模型权重

在128K上下文场景下，KV Cache（3,200GB）是模型权重（70GB）的45倍。这意味着HBM的需求主导项不是"模型有多大"，而是"上下文有多长、并发有多高"——两个变量都在快速增长。

06 · 核心结论

三个投资级别的确定性判断

这不是押注AI应用谁能赢，而是押注稀缺生产资料的供给方。AI应用竞争格局未定，但无论谁赢，都要消耗HBM。

结论一：HBM是卡脖子的那个
KV Cache在HBM中实时读写的带宽需求（TB/s级）是DDR物理上无法满足的——不是成本问题，是物理约束。Claude Sonnet 4.6在200K上下文、1000并发的企业部署中，KV Cache需求高达50TB HBM，单节点成本约$2.5亿。HBM的单位经济价值密度是LPDDR的约500倍（$5,000/GB vs $10/GB）。
结论二：上下文长度是HBM需求的乘法器
从32K→1M上下文，HBM的KV Cache需求增长31倍。Claude Sonnet 4.6的1M token beta功能一旦规模化，单次请求KV Cache需求达250GB，100并发即需25TB HBM——这是一个非线性跃升，而非线性扩展。
结论三：LPDDR是受益者而非驱动者
LPDDR5/DDR5的增长是被动的、跟随性的——因HBM溢出（MoE专家offload、超长上下文冷cache）而增长，并非因自身不可替代性。传统DRAM市场预计仅以3% CAGR增长至2030年，与HBM的40% CAGR形成鲜明对比。在当前时间点，HBM供应链（尤其是SK海力士+台积电CoWoS）是这个逻辑下最确定性的投资标的。

最终判断

这一系列分析的核心洞见高度成立，前瞻性强："唯一限制这个差距拉大的，只有产能扩充的速度"——这直接指向了投资机会的本质。HBM的供需紧张程度远超LPDDR5，需求曲线的斜率根本不在一个维度上，而产能扩充速度两者都受限于先进封装产能（CoWoS），这才是真正的瓶颈，也是真正的投资护城河所在。
